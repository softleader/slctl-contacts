# 規格書：Go 1.24.x 升級與 Go Modules 遷移

## 1. 概述
本 Track 旨在將 `slctl-contacts` 項目從過時的 Go 1.12.x 與 Glide 依賴管理系統，升級至現代的 Go 1.24.x 並採用 Go Modules。這將確保專案符合現代開發標準，提升編譯安全與維護性。

## 2. 功能需求
- **Go 版本升級**：將 `go.mod` (待建立) 指向 Go 1.24.x。
- **依賴管理系統遷移**：
    - 移除 `glide.yaml` 與 `glide.lock`。
    - 移除 `vendor/` 目錄，改用 Go Modules 標準管理方式。
    - 透過 `go mod init` 初始化專案模組。
- **套件全面升級**：
    - 將所有核心程式庫（如 `cobra`, `uitable`, `resty`）升級至最新的穩定版本。
    - 解決因套件升級而產生的語法不相容問題（如 `resty` 從 v1 到 v2 的遷移）。

## 3. 非功能需求
- **現代化實踐**：確保代碼符合 Go 1.24+ 的慣用法。
- **建置穩定性**：確保 `Makefile` 的建置流程能與 Go Modules 兼容。

## 4. 驗證準則 (Acceptance Criteria)
- [ ] 成功建立 `go.mod` 並設定 Go 1.24。
- [ ] `go build` 指令能順利完成編譯，無錯誤。
- [ ] 執行現有的單元測試 (`go test ./...`) 全部通過。
- [ ] 產生的二進位檔能正常啟動並顯示版本資訊。

## 5. 出範疇 (Out of Scope)
- 新增功能需求。
- 大規模的架構重構（除非是為了解決套件升級導致的相容性問題）。
