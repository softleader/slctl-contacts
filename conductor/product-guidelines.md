# 產品風格與設計規範 - slctl-contacts

## 1. 介面與互動設計 (UI/UX)
- **視覺呈現**：採用**豐富互動風格**。在終端機輸出時，視情況使用色彩區分不同欄位（如姓名、電話、部門），或使用 ASCII 圖標增強可讀性。
- **易讀性**：表格應具備適當的間距，確保在不同解析度的終端機中都能清晰顯示。
- **使用者回饋**：當查詢無結果時，應提供友善的提示資訊，而非僅顯示空白。

## 2. 文檔與溝通語言
- **語言規範**：本專案的程式碼註解、README 文檔以及 CLI 內的說明訊息，應優先使用**繁體中文**，以確保 SoftLeader 內部成員溝通無礙且直觀。

## 3. 開發規範
- **穩定性**：所有的通訊錄查詢應有超時 (Timeout) 處理，避免因網路延遲導致 CLI 程式長時間卡死。
- **一致性**：命令列參數的命名應符合 `slctl` 其他插件的既定慣例。
